About Me

Background

"Go Where" is my conceptual application that will help users easily discover new places to travel to, events to participate in, or activities to do.

My Role

• UX / UI designer

• UX reseacher

Year

2022

Problem

Finding new activities has become much more difficult.

According to research on digitalinformationworld.com, people's attention spans are shortening as mobile applications become more prevalent. Often, people want to have fun with their friends but don't know where to go or what to do. Searching for them is time-consuming and even more annoying when you have a short attention span, so they return to the same place over and over.

Competitive audit

No app on the market directly solves this problem

I looked for an application that would directly solve this problem, but I couldn't find one. Google Maps, one of the main indirect competitors' applications, is good at navigation but still not so good at place suggestion.

Understanding the users

Creating Persona

I interviewed five of my friends and they all have the same problem, so I created this persona to represent them.

Ideation

Focus on ease of use

I created the main application interface using the Crazy 8 ideation method. The main priority is usability, so I make the wireframe so that users must make as few decisions as possible.

Testing & Resign

Reduce Complexity

I conducted a moderated usability study with five of my friends. The majority of users (four out of five) believe the app will be helpful in real life, but the main page is still a little confusing. To fix this, I reduced the number of main action buttons to two, giving users only the choice of being interested in the place or not.

Final Product

What I learned

Designing process is not everything

I realize I wasted so much time on some designing processes that were not actually beneficial to the project. I complete it just to check the box, so I learn that you can skip some design processes as long as they do not impact the outcome of the project.
